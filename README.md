# 毕业设计：物联网设备安全性漏洞检测系统设计与实现

# 项目概述

本项目为毕业设计作品，核心目标是设计并实现一套物联网设备安全性漏洞检测系统。系统创新性融合传统静态分析技术与大语言模型（LLM），构建两阶段漏洞检测体系，解决传统漏洞检测中自动化程度低、语义分析不足、漏洞误报漏报率高的问题，实现物联网固件的全流程自动化漏洞检测与结构化报告输出，为物联网设备安全防护提供技术支撑。

# 核心功能

* **物联网固件自动化解析**：支持各类主流物联网设备固件的自动解析、提取与预处理，无需人工干预，快速获取固件中的二进制代码、配置文件等核心数据，为后续漏洞检测奠定基础。
* **复杂代码语义分析**：结合静态分析技术与大语言模型的语义推理能力，深入分析固件二进制代码的逻辑结构、函数调用关系，精准识别代码中的潜在安全隐患，突破传统静态分析对复杂语义理解不足的局限。
* **漏洞精准检测**：构建两阶段检测流程，第一阶段通过传统静态分析规则筛选高危可疑点，第二阶段通过大语言模型进行深度语义验证，大幅提升漏洞检测的准确性，降低误报、漏报率。
* **结构化漏洞检测报告输出**：检测完成后，自动生成包含漏洞位置、漏洞类型、风险等级、修复建议等信息的结构化报告，清晰直观，便于开发人员快速定位并修复漏洞。

# 技术栈与实现细节

## 1\. 固件解析与二进制处理

基于 **Binwalk** 和 **Ghidra** 两大核心框架，实现物联网固件的全流程自动化处理：

* Binwalk：负责固件的解包、提取，快速分离固件中的文件系统、二进制可执行文件等关键组件，解决不同格式固件的兼容性问题。
* Ghidra：对提取的二进制代码进行反汇编、反编译，生成结构化的代码逻辑（如函数列表、控制流图），为后续静态分析和语义分析提供标准化输入。

## 2\. 传统静态分析体系构建

自主构建污点分析规则引擎与高危函数特征库，夯实第一阶段漏洞筛选能力：

* 污点分析规则引擎：定义一套完整的污点传播规则，追踪用户输入、外部数据在代码中的流转路径，识别因输入验证不足、权限管控缺失导致的漏洞（如缓冲区溢出、命令注入等）。
* 高危函数特征库：收集物联网设备固件中高频出现的高危函数（如字符串拷贝函数、系统调用函数等），建立函数特征模板，快速匹配代码中的高危调用场景，实现初步漏洞筛选。

## 3\. 大语言模型融合与优化

引入 **CodeBERT**、**DeepSeek** 等大语言模型，通过定制化优化，突破传统静态分析瓶颈，构建混合式检测体系：

* 模型选型：选用CodeBERT（擅长代码语义理解）、DeepSeek（高效推理能力）作为核心模型，兼顾代码分析的准确性与检测效率。
* 定制化Prompt模板：针对物联网固件代码特点，设计专属Prompt模板，引导模型聚焦漏洞检测场景，提升语义分析的针对性和准确性。
* 数据集微调：基于物联网固件漏洞数据集进行模型微调，让模型适配物联网设备代码的语法、逻辑特征，进一步降低漏洞误报、漏报率。

## 4\. 混合式检测体系

整合传统静态分析与大语言模型语义推理，形成两阶段检测流程：

1. 第一阶段（规则筛选）：通过污点分析规则引擎和高危函数特征库，快速筛选出代码中的可疑漏洞点，减少后续模型分析的计算量，提升检测效率。
2. 第二阶段（语义验证）：将可疑漏洞点输入经微调的大语言模型，通过模型的语义推理能力，验证漏洞的真实性，精准判断漏洞类型与风险等级，输出最终检测结果。

# 项目意义

随着物联网设备的普及，固件漏洞已成为物联网安全的核心隐患。本系统通过融合传统静态分析与大语言模型的优势，实现物联网固件漏洞的自动化、精准化检测，解决了传统检测方法效率低、语义理解不足的痛点，可为物联网设备厂商、安全检测机构提供高效的漏洞检测工具，助力提升物联网设备的整体安全性，具有较强的理论价值和实际应用意义。

# 备注

本项目为毕业设计作品，聚焦物联网设备固件漏洞检测的核心需求，后续可进一步优化模型性能、扩展支持的固件类型，完善漏洞修复建议库，提升系统的实用性和扩展性。


# 项目一：基于Transformer的恶意加密流量检测

# 项目概述

本项目聚焦加密流量安全检测领域，设计并实现了一套基于Transformer模型的恶意加密流量检测系统。针对当前加密流量（如HTTPS、SSH）难以有效区分正常与恶意行为的痛点，借助Transformer模型的强大特征提取与序列分析能力，实现对常见加密协议流量的快速识别、分类，精准甄别正常加密流量与恶意加密流量，为网络安全防护提供高效、可靠的技术支撑，可应用于企业内网安全监测、网络入侵检测等场景。

# 核心功能

* **加密协议流量识别**：支持对HTTPS、SSH等主流加密协议的流量进行自动识别，无需人工干预，可快速区分不同类型的加密流量，适配多种网络场景下的流量检测需求。
* **流量分类与恶意检测**：核心实现正常加密流量与恶意加密流量的精准分类，能够有效识别隐藏在加密流量中的恶意行为（如恶意攻击、数据窃取等），降低加密流量带来的安全隐患。
* **高效检测性能**：基于Transformer模型的优化设计，兼顾检测速度与准确率，可实现加密流量的快速检测，满足实时网络监测的性能要求。

# 个人负责部分

作为项目核心参与者，主要承担以下工作内容，助力项目高效落地：

* **Transformer模型调优与训练**：参与Transformer模型的结构设计与参数调优，结合加密流量数据的特征，优化模型训练策略，提升模型的收敛速度与检测性能；负责模型的全流程训练，包括训练数据划分、训练过程监控与参数迭代优化。
* **性能验证与指标分析**：主导项目的性能验证工作，设计合理的验证方案，重点分析模型检测准确率、召回率、F1值等关键评价指标，量化模型检测效果；针对验证过程中发现的问题，提出针对性优化建议，推动模型性能迭代提升。

# 技术栈与实现细节

## 1\. 编程语言与核心框架

* **Python**：作为项目主要编程语言，负责整体代码开发、数据处理、模型搭建与性能验证，具备简洁、高效、易扩展的优势，适配数据处理与机器学习开发场景。
* **PyTorch**：作为核心深度学习框架，用于搭建Transformer模型，实现模型的构建、训练与推理；借助PyTorch的动态计算图特性，便于模型参数调优与训练过程的灵活调整，提升开发效率。

## 2\. 数据处理与特征工程

采用Scikit-learn工具库完成数据预处理与特征工程，为模型训练提供高质量输入：

* 数据预处理：对原始加密流量数据进行清洗、去噪、归一化处理，处理缺失值与异常值，确保数据的完整性与有效性；将流量数据转换为模型可识别的序列格式，适配Transformer模型的输入要求。
* 特征工程：基于加密流量的特征（如数据包长度、交互频率、协议字段等），提取关键特征，构建有效的特征向量；通过特征选择、特征变换等操作，提升特征的区分度，助力模型精准学习正常与恶意流量的差异。

## 3\. 模型训练可视化

搭配TensorBoard工具，实现模型训练过程的可视化监控：

* 实时监控训练过程中的损失值、准确率等关键指标，直观观察模型的收敛情况，及时发现训练过程中的问题（如过拟合、欠拟合）。
* 可视化模型结构、特征分布等，便于分析模型性能瓶颈，为模型调优提供直观依据，提升模型优化的效率与针对性。

# 项目意义

当前加密技术的广泛应用，既保障了数据传输的安全性，也为恶意流量提供了隐藏渠道，传统流量检测方法难以有效识别加密流量中的恶意行为。本项目基于Transformer模型构建恶意加密流量检测系统，突破传统检测方法的局限，实现对加密流量的精准检测，能够帮助相关机构快速发现网络中的恶意行为，降低网络安全风险，具有重要的实际应用价值与研究意义。

# 备注

本项目聚焦恶意加密流量检测的核心需求，后续可进一步扩展支持的加密协议类型，优化模型结构以提升检测速度与泛化能力，完善数据集覆盖范围，增强系统在复杂网络环境下的适用性。


# 项目二：同态加密隐私保护系统的设计与实现

# 项目概述

本项目聚焦数据隐私保护领域，设计并实现了一套基于同态加密技术的隐私保护系统。针对当前数据共享与处理过程中存在的隐私泄露风险，借助同态加密的核心特性，实现“数据加密后仍可运算”的核心目标，在不解密原始敏感数据的前提下，完成对加密数据的各类运算处理，既保障数据隐私安全，又实现数据的安全共享与高效利用，可广泛应用于医疗数据共享、金融隐私计算、政务数据协同等对数据隐私要求极高的场景。

# 核心功能

* **同态加密核心运算支持**：系统核心支持在不解密原始数据的前提下，对加密数据执行加法、乘法等基础运算，同时可拓展支持复杂组合运算，确保数据在全生命周期内处于加密状态，从根源上杜绝数据隐私泄露。
* **全流程数据安全处理**：实现数据从加密上传、运算处理到解密输出的全流程闭环功能，无需人工干预数据解密过程，降低人为操作带来的隐私泄露风险，保障数据处理的安全性与规范性。
* **多算法适配**：集成BGV、BFV、CKKS三种主流同态加密算法，可根据不同数据类型、运算需求灵活选择适配的算法，兼顾运算效率与隐私保护强度，适配多种实际应用场景。
* **便捷交互操作**：搭建友好的系统交互界面，简化用户操作流程，用户可轻松完成数据加密上传、运算参数设置、运算结果查看与解密输出等操作，降低系统使用门槛。

# 个人负责部分

作为项目核心参与者，深度参与系统全流程开发与设计，主要承担以下关键工作：

* **系统整体架构设计**：参与系统整体架构的规划与设计，结合同态加密技术特性与实际应用需求，搭建清晰、可扩展的系统架构，明确各模块的功能边界、交互逻辑与数据流转路径，保障系统的稳定性与可扩展性。
* **同态加密算法研究与实现**：主导同态加密算法的调研、研究与落地实现，深入研究BGV、BFV、CKKS三种主流同态加密算法的原理与特性，结合项目需求完成算法的代码实现与优化，解决算法实现过程中的效率与安全性问题。
* **加密与解密模块开发**：负责数据加密、解密核心模块的设计与开发，封装加密解密接口，实现原始数据的高效加密、加密数据的安全解密功能，确保加密解密过程的安全性、高效性，适配系统全流程数据处理需求。

# 技术栈与实现细节

## 1\. 编程语言与核心库

* **编程语言**：采用C++与Python混合编程模式，兼顾系统性能与开发效率。C++主要用于同态加密算法的底层实现，保障算法运行效率与安全性；Python用于系统交互界面开发、数据处理及模块间的交互调度，提升开发效率与系统灵活性。
* **核心库**：基于Microsoft SEAL（Simple Encrypted Arithmetic Library）库实现同态加密算法，该库是微软开源的高性能同态加密库，支持BGV、BFV、CKKS等主流同态加密算法，提供完善的API接口，大幅降低算法实现难度，提升系统开发效率与稳定性。

## 2\. 同态加密算法实现

基于Microsoft SEAL库，完成三种主流同态加密算法的集成与实现，适配不同场景需求：

* BGV算法：主要用于整数域上的同态运算，适配对整数数据进行加密运算的场景，具备较高的运算效率与安全性，适用于简单数值型数据的隐私计算。
* BFV算法：针对整数多项式环设计，支持更高效的同态加法与乘法运算，适合需要频繁执行基础运算的场景，提升系统运算效率。
* CKKS算法：支持实数域上的同态运算，可处理浮点数数据，适用于需要对连续型数据（如医疗指标、金融数据）进行隐私计算的场景，拓展系统的应用范围。

## 3\. 系统交互界面与全流程实现

搭建直观、便捷的系统交互界面，串联数据加密、运算处理、解密输出全流程，实现端到端的隐私保护数据处理：

* 数据加密上传：用户通过界面上传原始敏感数据，系统自动调用加密模块完成数据加密，生成加密数据并存储，确保原始数据不泄露。
* 运算处理：用户在界面设置运算类型（加法、乘法等），系统调用对应同态加密算法对加密数据执行运算，运算过程全程不解密原始数据，保障数据隐私。
* 解密输出：运算完成后，用户通过界面提交解密请求，系统调用解密模块对运算结果进行解密，输出原始运算结果，确保结果的准确性与安全性。

# 项目意义

在数据隐私保护日益重要的当下，传统数据处理模式需解密原始数据才能进行运算，极易导致隐私泄露。本项目基于同态加密技术，突破传统数据处理的隐私瓶颈，实现“加密后运算”的核心能力，既保障了敏感数据的隐私安全，又实现了数据的安全共享与高效处理，有效解决了数据隐私保护与数据利用之间的矛盾。项目的实现可为隐私计算领域提供实用的技术方案，推动同态加密技术在各行业的落地应用，具有重要的理论研究价值与实际应用意义。

# 备注

本项目已完成核心功能的开发与测试，后续可进一步优化同态加密算法的运算效率，拓展更复杂的运算类型，完善交互界面的用户体验，同时扩大数据集适配范围，提升系统在复杂场景下的适用性与稳定性。


# 项目三：大型语言模型水印技术的检测与取证研究

# 项目概述

本项目聚焦于大型语言模型（LLM）水印技术的检测与取证领域，开展系统性的研究工作，核心目标是通过技术手段实现对LLM生成内容中隐含水印的识别、溯源与版权取证，进而防范模型滥用及生成内容伪造带来的各类风险，为LLM技术的合规、安全应用提供技术支撑。

# 主要功能

* 开展大型语言模型水印嵌入算法与水印检测算法的全面调研，梳理现有技术的优势与不足，形成系统性的技术调研报告。
* 实现对LLM生成内容中隐含水印的精准识别，突破水印检测过程中的噪声干扰、水印嵌入强度低等技术难点。
* 完成LLM生成内容的溯源功能，通过水印信息定位生成内容对应的模型来源，实现生成内容的可追溯。
* 提供生成内容的版权取证技术支持，通过水印检测结果固化版权证据，为版权纠纷处理提供技术依据。
* 针对模型滥用、内容伪造等问题，基于水印检测与取证技术，形成针对性的防范方案，提升LLM应用的安全性与合规性。

# 负责部分

* 负责大型语言模型（LLM）的全面调研与测试工作，包括主流LLM的特性、水印嵌入方式、生成内容特征等，建立完善的模型测试数据集与测试标准。
* 负责实验方案的设计、实验过程的执行，精准记录实验数据，撰写详细的实验报告，包括实验目的、实验环境、实验步骤、实验结果及分析等内容。
* 负责研究过程的总结梳理，整合调研成果、实验结论，形成完整的研究总结报告，提炼核心技术要点与研究创新点，为项目后续优化提供方向。

# 技术运用

* **编程语言**：采用Python作为核心编程语言，依托其丰富的开源库与高效的开发效率，实现水印算法的研究、复现与检测逻辑的开发。
* **核心技术**：

  * 自然语言处理（NLP）：运用NLP相关技术，对LLM生成内容进行预处理、特征提取，挖掘与水印相关的文本特征。

# 远程屏幕监控系统 README 总结

## 项目概述

基于Java开发的C/S架构远程屏幕监控系统，实现客户端屏幕实时采集、加密传输、服务器端存储与展示，配套用户认证、截屏频率调整等功能，兼顾安全性与可扩展性。

## 核心信息

### 1\. 功能核心

- 客户端：屏幕实时截屏、系统托盘常驻、登录注册、与服务器连接/断开

- 服务器端：多客户端并发支持、屏幕实时展示、操作日志记录、截屏存储、截屏频率动态调整

- 安全与存储：SSL/TLS加密通信、SQL Server数据库存储用户及截屏数据、本地目录备份截屏

### 2\. 技术栈

开发语言：Java \(JDK 17\+\)；网络通信：Socket \+ SSL/TLS；数据库：SQL Server；GUI框架：Swing；构建工具：IntelliJ IDEA；加密方式：JKS密钥库/信任库

### 3\. 环境与配置

- 基础环境：JDK 17\+、SQL Server 2019\+、可互通网络、Java IDE

- 数据库：创建CStest数据库，建立users（用户表）和screenshots（截屏表），配置授权账号

- SSL配置：服务器/SERVER目录、客户端/CLIENT目录分别放置对应JKS证书，默认密码可修改

### 4\. 快速启动

1. 服务器：配置数据库连接参数，运行server\.ServerMain，等待客户端连接

2. 客户端：配置服务器地址、数据库参数，运行client\.ClientMain，完成登录注册后连接服务器

### 5\. 关键注意事项

- 安全：保护SSL证书，修改默认密码，限制数据库账号权限

- 性能：截屏频率\&lt;500ms可能影响客户端性能，多客户端并发需提升服务器资源

- 兼容性：系统托盘依赖AWT，部分Linux环境不支持；SSL证书需客户端与服务器匹配

- 异常：数据库连接失败影响登录功能，端口占用需修改端口重启

### 6\. 其他

待优化：截屏压缩、权限管理、历史回放、Web端监控等；项目仅供学习研究，禁止商业/非法用途，开发者不承担滥用后果。
