# 远程屏幕监控系统 README 总结

## 项目概述

基于Java开发的C/S架构远程屏幕监控系统，实现客户端屏幕实时采集、加密传输、服务器端存储与展示，配套用户认证、截屏频率调整等功能，兼顾安全性与可扩展性。

## 核心信息

### 1\. 功能核心

- 客户端：屏幕实时截屏、系统托盘常驻、登录注册、与服务器连接/断开

- 服务器端：多客户端并发支持、屏幕实时展示、操作日志记录、截屏存储、截屏频率动态调整

- 安全与存储：SSL/TLS加密通信、SQL Server数据库存储用户及截屏数据、本地目录备份截屏

### 2\. 技术栈

开发语言：Java \(JDK 17\+\)；网络通信：Socket \+ SSL/TLS；数据库：SQL Server；GUI框架：Swing；构建工具：IntelliJ IDEA；加密方式：JKS密钥库/信任库

### 3\. 环境与配置

- 基础环境：JDK 17\+、SQL Server 2019\+、可互通网络、Java IDE

- 数据库：创建CStest数据库，建立users（用户表）和screenshots（截屏表），配置授权账号

- SSL配置：服务器/SERVER目录、客户端/CLIENT目录分别放置对应JKS证书，默认密码可修改

### 4\. 快速启动

1. 服务器：配置数据库连接参数，运行server\.ServerMain，等待客户端连接

2. 客户端：配置服务器地址、数据库参数，运行client\.ClientMain，完成登录注册后连接服务器

### 5\. 关键注意事项

- 安全：保护SSL证书，修改默认密码，限制数据库账号权限

- 性能：截屏频率\&lt;500ms可能影响客户端性能，多客户端并发需提升服务器资源

- 兼容性：系统托盘依赖AWT，部分Linux环境不支持；SSL证书需客户端与服务器匹配

- 异常：数据库连接失败影响登录功能，端口占用需修改端口重启

待优化：截屏压缩、权限管理、历史回放、Web端监控等；项目仅供学习研究，禁止商业/非法用途，开发者不承担滥用后果。
